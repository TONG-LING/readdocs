实施建议
========

当前问题
--------

标签差异太集中   大部分 `balance_score` 集中在较窄区间，优秀样本之间差距小。

差样本拉大波动   少量较差样本会拉大整体标准差，模型容易只学到明显差样本。

预测误差偏大   验证 MAE 接近真实标签标准差，预测误差足以打乱候选排序。

排序能力不足   后验筛选依赖相对排序，但当前 Spearman 相关接近 0。

样本来源有偏   训练样本主要来自当前进化过程，对新候选池的覆盖可能不足。

改进方向
--------

改用排序学习   后验筛选本质是选择相对更好的候选，可以训练模型直接比较两个样本谁更好。

多目标拆分预测   分别预测 `q`、`worst_dir_queue`、`dir_spread`、`q_balance_penalty`，再组合成最终评分。

优化样本采样   在优秀样本附近增加局部扰动样本，提高模型对高质量区域的区分能力。

保留不确定样本   对代理模型不确定但可能较好的样本保留一定比例送入 SUMO，避免过早过滤潜在优秀样本。

增加候选池质量评估   单独检查扩散模型候选池中是否真的存在高质量样本。

增加随机筛选对照   对比代理后验筛选和随机筛选的真实 SUMO 表现，判断代理模型是否带来额外收益。

下一步实验
----------

建议优先补充三个实验：

.. code-block:: text

   1. 代理预测排序与真实排序的 Spearman 统计
   2. 扩散候选池真实上限评估
   3. 代理筛选与随机筛选对照实验

如果候选池中存在真实好样本，但代理模型没有选中，说明问题主要在代理模型。

如果候选池中本身缺少好样本，说明扩散模型生成质量也需要继续改进。
